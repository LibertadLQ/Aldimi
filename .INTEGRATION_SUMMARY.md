**Status**: ✅ **Integration Complete and Ready for Testing**

## Overview

This document summarizes the complete integration of alert detection, clinical suggestions, and laboratory data persistence into the ALDIMI chatbot system. All changes have been made to ensure that:

1. **Lab data is always available** in patient expedientes
2. **Alerts are detected and enriched** with severity levels and care suggestions
3. **The chatbot surfaces** alerts and recommendations to users
4. **End-to-end workflow** can be activated with `run.ps1`

---

## Key Components

### 1. **Alert Evaluation Module** (`backend/alertas.py`)

**Purpose**: Rule-based evaluation of laboratory test results to generate structured alerts with severity levels and clinical suggestions.

**Key Functions**:
- `evaluar_prueba(prueba)` — Evaluates a single lab test and returns an enriched alert
- `evaluar_pruebas(pruebas)` — Batch evaluation of multiple lab tests
- `filtrar_alertas_criticas(registro, ciu)` — Formats critical alerts into a user-friendly response

**Alert Output Structure**:
```python
{
    "timestamp": "2025-01-15T10:30:00",
    "prueba": "Hemoglobina",
    "valor": 6.5,
    "unidad": "g/dL",
    "tipo": "BAJO",
    "severity": "critical",  # critical, high, medium, low
    "reason": "default_ref",
    "title": "Anemia Crítica",
    "suggestion": "Reposo absoluto, control de signos vitales y posible transfusión. Contactar equipo médico.",
    "source": "ocr"
}
```

**Reference Ranges** (with severity rules for):
- Hemoglobina (Hb): critical <7, high <10
- Plaquetas: critical <20k, high <50k
- Neutrófilos: critical <500, high <1.5k
- Potasio (K): high if <3.5 or >5.5
- Creatinina: high if >1.5
- Proteína C Reactiva (CRP): high if >50

### 2. **OCR Pipeline Enrichment** (`backend/ocr_robusto.py`)

**Changes**:
- Post-detection enrichment: After detecting alerts via regex/flags, the OCR parser now calls `alertas.evaluar_pruebas()` to enhance each alert with severity and suggestions
- Merges evaluator output into existing alerts and appends any missing alerts found only by the evaluator
- Maintains backward compatibility with existing alert detection flow

**Integration Point**:
```python
# In procesar_lab():
alertas_detectadas = detectar_alertas(...)  # Existing regex-based detection
# NEW: Enrich with severity and suggestions
alertas_enriquecidas = evaluar_pruebas([...])
# Merge: existing + new
alertas_finales = merge_alerts(alertas_detectadas, alertas_enriquecidas)
```

### 4. **Chatbot Alert Handling** (`backend/chatbot.py`)

**New Features**:
- **Intent**: `alertas_clinicas` — User can ask "Show me clinical alerts" or similar
- **Function**: `filtrar_alertas_criticas()` — Formats only critical/alert-flagged tests into readable response
- **Fallback**: If no alerts, returns "✅ No alerts detected" message
- **Format**: 
  ```
  🚨 Alertas clínicas — CIU 42951703
  Se detectaron 3 valores fuera de rango:
  🔺 ALTO: Hemoglobina = 6.5 g/dL (Ref: 12-17.5)
  🔺 ALTO: Plaquetas = 25000 /µL (Ref: 150-450k)
  🔻 BAJO: Potasio = 3.2 mmol/L (Ref: 3.5-5.2)
  ```

**Integration Points**:
- Imported: `from .alertas import filtrar_alertas_criticas`
- Used in: `procesar_mensaje()` when intent = `"alertas_clinicas"`
- State management: `pedir_ciu_alertas` — waits for user to provide CIU before fetching alerts

**Recommendations**:
- Function `generar_recomendaciones(registro)` generates care suggestions based on detected alerts
- Example: "Hemoglobina: evaluar signos de anemia y considerar reforzar la dieta con hierro."

### 5. **Startup Script** (`run.ps1`)

**Configuration** (already set):
```powershell
$env:ALDIMI_AUTO_SCAN = "true"           # Enable background scan
$env:ALDIMI_WAIT_FOR_SCAN = "false"      # Don't block on startup
$env:ALDIMI_SCAN_DNI = "100"             # Scan up to 100 DNI images
$env:ALDIMI_SCAN_LAB = "100"             # Scan up to 100 lab images
```

**Workflow**:
1. Creates/activates Python venv
2. Installs dependencies from `backend/requirements.txt`
3. Checks for Tesseract OCR (auto-install on Windows if needed)
4. Starts uvicorn backend (port 8000) in new PowerShell window
5. Starts static HTTP server (port 5500) in new PowerShell window
6. Waits for backend readiness (with retries)
7. Opens `http://localhost:5500/chatbot.html` in browser

**Autoscan Behavior**:
- Runs in background (non-blocking) when server starts
- FastAPI `@app.on_event("startup")` triggers folder scan
- Loads existing patient data from JSON DB
- Processes any new OCR images in `DNI_ALDIMI/` and `LAB_ALDIMI/` directories

## End-to-End Workflow

### User Interaction Flow

1. **User opens chatbot.html** (after `run.ps1` starts)
   - Frontend connects to `/ready` endpoint
   - Backend loads DB and autoscan begins

2. **User enters CIU** to view expediente or check alerts
   - Frontend sends to `/chat` with intent
   - Chatbot processes and calls `filtrar_alertas_criticas()` if alerts requested
   - Backend returns formatted alert list with severity and suggestions

3. **Backend processes new lab images** (if in LAB_ALDIMI folder)
   - OCR extracts test results
   - `backend/alertas.py` evaluates and enriches with severity
   - Alert objects stored in `aldimi_pacientes.json`

4. **User queries patient evolution or recommendations**
   - Chatbot accesses `informes_laboratorio` (always present)
   - Compares lab trends over time
   - Generates personalized recommendations based on active alerts

## Support & Troubleshooting

### Backend won't start
- Check `backend/backend.log` for import errors
- Verify all dependencies: `pip install -r backend/requirements.txt`
- Confirm Tesseract OCR installed: `tesseract --version`

### Alerts not appearing
- Check that patient has `informes_laboratorio` with test data
- Verify alert request goes to `/chat` with `intent=alertas_clinicas`
- Check browser console for API errors

### Lab images not auto-scanning
- Verify images are in correct folders: `DNI_ALDIMI/` or `LAB_ALDIMI/`
- Check that `ALDIMI_AUTO_SCAN=true` in `run.ps1`
- Review server startup logs for scan completion message
